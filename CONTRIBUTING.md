## 行動規範

このプロジェクトでは、誰もが歓迎され、尊重される環境を大切にしています:

- 敬意を持ったコミュニケーション
- 建設的なフィードバック
- 他者の意見を尊重

## 開発環境のセットアップ

### 必要なもの

- **Firefox** (最新版推奨)
- **Anki** (デスクトップ版)
- **AnkiConnect** アドオン (コード: `2055492159`)
- **Git**
- **テキストエディタ** (VS Code, Vim, etc.)

### セットアップ手順

1. **リポジトリをフォーク**

   GitHubでこのリポジトリをフォークしてください。

2. **クローン**

   ```bash
   git clone https://github.com/yourusername/anki-word-adder.git
   cd anki-word-adder
   ```

3. **拡張機能を読み込む**

   - Firefoxで `about:debugging#/runtime/this-firefox` を開く
   - **「一時的なアドオンを読み込む」** をクリック
   - `manifest.json` を選択

4. **Ankiを起動**

   AnkiConnectが `localhost:8765` で動作していることを確認。

5. **開発開始！**

   コードを編集後、拡張機能をリロードして変更を確認。

## コーディング規約

### JavaScript

- **インデント**: 2スペース
- **セミコロン**: 使用する
- **命名規則**:
  - 関数: `camelCase` (例: `fetchTranslation`)
  - 定数: `UPPER_SNAKE_CASE` (例: `DEFAULT_SETTINGS`)
  - 変数: `camelCase`
- **コメント**: 日本語でOK（可読性優先）
- **非同期処理**: `async/await` を優先（Promiseチェーンより読みやすい）

### HTML/CSS

- **インデント**: 2スペース
- **クラス名**: `kebab-case` (例: `modal-overlay`)
- **ID**: `camelCase` (例: `wordInput`)

### コミットメッセージ

明確で説明的なコミットメッセージを書いてください:

```
Add dictionary API integration for English words

- Fetch pronunciation, definitions, and examples from Free Dictionary API
- Display dictionary data in modal
- Add useDictionary setting to options page
```

**推奨フォーマット:**
- 1行目: 変更の概要（50文字以内、英語または日本語）
- 空行
- 詳細な説明（必要に応じて）

## プルリクエストの流れ

1. **イシューを確認・作成**

   既存のイシューがあるか確認。なければ新しいイシューを作成して、実装内容を議論してください。

2. **ブランチを作成**

   ```bash
   git checkout -b feature/your-feature-name
   ```

   ブランチ名の例:
   - `feature/add-image-capture`
   - `fix/duplicate-error-handling`
   - `docs/update-readme`

3. **変更を実装**

   - コーディング規約に従う
   - 既存の機能を壊さない
   - 可能な限りシンプルに保つ

4. **テスト**

   [README.md](README.md) のテストチェックリストを実行:
   - 基本機能が動作する
   - エラーハンドリングが適切
   - 設定が保存される
   - 新機能が意図通り動作する

5. **コミット**

   ```bash
   git add .
   git commit -m "Add feature: ..."
   ```

6. **プッシュ**

   ```bash
   git push origin feature/your-feature-name
   ```

7. **プルリクエストを作成**

   GitHub上でプルリクエストを作成:
   - **タイトル**: 変更の概要
   - **説明**: 変更内容、動機、テスト方法
   - **関連イシュー**: `Closes #123` のように記載

8. **レビュー対応**

   フィードバックに基づいて修正し、再度プッシュ。

## テストガイドライン

現在、自動テストは実装されていません。以下の手動テストを推奨します:

### 基本機能

- [ ] 英単語を選択 → 右クリックメニュー → モーダル表示
- [ ] 日本語を選択 → 自動翻訳が英語になる
- [ ] 翻訳フィールドを編集 → Ankiに反映される
- [ ] デッキ選択 → 指定デッキに追加される

### 新機能

- [ ] 辞書API: 英単語で発音・定義が表示される
- [ ] スクリーンショット: プレビューが表示される
- [ ] キーボードショートカット: `Ctrl+Shift+A` で動作する

### エラーハンドリング

- [ ] Anki未起動 → エラーメッセージ表示
- [ ] 重複単語 → 重複エラー表示
- [ ] ネットワークエラー → タイムアウトメッセージ

### 設定

- [ ] 設定変更 → 保存される
- [ ] リロード → 設定が保持される
- [ ] デフォルト値が正しい
